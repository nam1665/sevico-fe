# Contributing to Sevico

Thank you for considering contributing to Sevico! This document provides guidelines for contributing to the project.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/sevico-fe.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test your changes: `npm test`
6. Commit your changes: `git commit -m "Add your feature"`
7. Push to your fork: `git push origin feature/your-feature-name`
8. Open a Pull Request

## Development Guidelines

### Code Style

- Follow the existing code style
- Run `npm run format` before committing
- Ensure `npm run lint` passes without errors
- Use TypeScript for all new code

### Commit Messages

Use clear, descriptive commit messages:

- `feat: add new component`
- `fix: resolve login bug`
- `docs: update README`
- `test: add Button component tests`
- `refactor: simplify auth logic`

### Testing

- Write tests for new features
- Ensure all tests pass before submitting PR
- Aim for >80% code coverage

### Pull Requests

- Provide a clear description of changes
- Reference related issues
- Include screenshots for UI changes
- Ensure CI checks pass

## Questions?

Feel free to open an issue for any questions or concerns.
